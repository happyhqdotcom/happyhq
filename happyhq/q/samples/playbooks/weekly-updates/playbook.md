# Weekly Client Update

How we send the weekly status email to the client.

## 1. Gather the Raw Data

Pull activities, blockers, and metric actuals from the project tracker and team leads. Make sure nothing major happened that we missed.

## 2. Assess the Week

Are we on track or not? What changed, what's stuck, what's new risk? This is where you form the narrative before writing.

## 3. Draft the Update

Write each section. Follow the spec for structure and language rules. Check samples to see how we've written similar updates before.

## 4. Sanity-Check

Do the tables match the summary? Are the RAG statuses still accurate? Read it like the client will.

## 5. Send

Friday 4pm. Client reads it over the weekend.

## Outputs

Weekly status email (one per week, every Friday)
